# 運用ルール

TAKUMIとSORAが摩擦なく動くための約束事。

---

## TAKUMIのルール（3つだけ）

| ルール | 詳細 |
|---|---|
| **1. 迷ったら00_Inboxに入れる** | どのフォルダか迷う時間は無駄。全部Inboxに投げてSORAに整理させる |
| **2. タグを1つ付ける** | SORAへの依頼はタグで伝える。文章で説明しなくていい |
| **3. 完璧に書かない** | 箇条書き・断片・走り書きでOK。整えるのはSORAの仕事 |

---

## SORAタグ一覧（6つ）

| タグ     | SORAがやること               | 出力先                   |
| ------ | ----------------------- | --------------------- |
| `#調査`  | テーマをWeb検索して要約・箇条書きにまとめる | 同ノートに追記               |
| `#タスク` | 内容をタスクリストに分解する          | `10_Projects` に新ノート作成 |
| `#実行`  | 指示された作業をそのまま実行する        | `10_Projects` に成果物を配置 |
| `#作成`  | ファイル・資料・Excelなどを生成する    | `10_Projects` に成果物を配置 |
| `#整理`  | 要約・タグ付けして知識ノートに変換する     | `20_Resources` に移動    |
| `#メール` | 返信メールの下書きを作成する          | 同ノートに追記               |

---

## タグの使い分け早見表

| 状況 | 使うタグ |
|---|---|
| 「〇〇について調べたい」 | `#調査` |
| 「何をすべきか整理したい」 | `#タスク` |
| 「これをやってほしい（作業依頼）」 | `#実行` |
| 「ファイルや資料を作ってほしい」 | `#作成` |
| 「このメモを知識として保存したい」 | `#整理` |
| 「メールの返信を書いてほしい」 | `#メール` |

---

## フォルダの使い分け

| フォルダ | 何を入れるか | 誰が管理するか |
|---|---|---|
| `00_Inbox` | 全ての新規メモ・依頼 | TAKUMI（投げるだけ） |
| `10_Projects` | 進行中のプロジェクト・タスク・成果物 | SORA（タスク化・整理） |
| `20_Resources` | 恒久的な知識・参考情報 | SORA（整理・リンク付け） |
| `30_Archive` | 完了・不要になったもの | SORA（自動移動） |
| `99_System` | テンプレート・ルール・ダッシュボード | SORA（管理） |

---

## 日常の流れ（理想形）

```
朝：デイリーノートを開いて今日のフォーカスを1行書く
日中：気づいたことを00_Inboxに投げる（タグを付ける）
夜：Obsidian Gitが自動でGitHubにpush
翌朝：SORAが処理した結果がObsidianに反映されている
```
