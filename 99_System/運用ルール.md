# Obsidian × SORA 運用ルール (v05)

## 1. 基本方針
- **SORAアイデンティティ**: 戦略コンサル兼シニアPMとしての「SORA」が、全てのタスクをイシュー思考・ピラミッド原則に基づき処理する。
- **Inbox保持**: ユーザーが作成した元ノートは常に `00_Inbox` に保持し、SORAが移動させることはない。
- **自律的サイクル**: 平日2時間おきの定期実行により、デイリーノートの管理とタスク処理を自動化する。

## 2. フォルダ構成
- `00_Inbox`: TAKUMIのメモ・依頼の入り口。元ノートはここに永続保持される。
- `01_Daily`: 自動生成されるデイリーノート。
- `10_Projects`: 進行中のタスク・実行計画・成果物・インシデントレポート。
- `20_Resources`: 調査結果・整理済み知識・週次レビュー。
- `99_System`: 運用ルール・プロンプト・テンプレート。

## 3. 定期実行スケジュール (平日限定)
| 時刻 (JST) | 処理内容 |
|---|---|
| **08:00** | **デイリーノート生成** + 未処理タグの処理 |
| **10:00 - 18:00** | 未処理タグの処理（2時間おき） |
| **20:00** | **活動ログの自動記入** + 未処理タグの処理 |

## 4. タグと処理フロー（全9タグ）
| タグ | 役割 | 成果物出力先 |
|---|---|---|
| `#調査` | イベント報告・詳細レポート | `20_Resources` |
| `#タスク` | タスク分解・実行計画 | `10_Projects` |
| `#実行` | 資料作成・実務遂行 | `10_Projects` |
| `#作成` | ファイル・資料生成 | `10_Projects` |
| `#整理` | 思考整理・構造化 | `20_Resources` |
| `#メール` | エグゼクティブチャット | `00_Inbox` |
| `#指示` | 自由指示の実行 | 内容により判断 |
| `#緊急` | インシデント対応 | `10_Projects` |
| `#週次レビュー` | 週次の振り返り | `20_Resources` |

## 5. 処理済みマークとリンク
- **タグ隠蔽**: 処理済みのタグは `<!-- #タグ名 -->` に置換され、プレビューでは非表示になる。
- **双方向リンク**: 
  - 成果物ノートの冒頭に `source: [[../00_Inbox/元ノート名]]` を記載。
  - 元ノートの末尾に `成果物: [[成果物名]]` を記載。

## 6. 命名規則
- **デイリーノート**: `YYYY-MM-DD.md`
- **成果物ファイル名**: `YYYY-MM-DD_元ノート名_タグ名_v01.md`
- （例: `2026-05-13_モダナイ準備_整理_v01.md`）
