# IT-0914 Obsidian Vault

TAKUMIの個人用Obsidian Vaultであり、AIエージェント「SORA」との共有ワークスペースです。

## 概要

このVaultは「PARAメソッド」をベースにした5フォルダ構成と、SORAへの依頼をトリガーする「6つのタグ」によって運用されます。TAKUMIがメモを書いてタグを付け、GitHubにpushすることで、SORAが自律的にタスクを処理し、結果をこのVaultに書き戻します。

## フォルダ構成

| フォルダ | 役割 | 管理者 |
|---|---|---|
| `00_Inbox` | 全ての新規メモ・SORAへの依頼の入り口 | TAKUMI |
| `10_Projects` | 進行中のプロジェクト・タスク・生成された成果物 | SORA |
| `20_Resources` | 恒久的な知識ノート・参考情報 | SORA |
| `30_Archive` | 完了したプロジェクト・不要になったノート | SORA |
| `99_System` | テンプレート・運用ルール・ダッシュボード | SORA |

## SORAタグ一覧（6タグ）

SORAへの依頼は以下のタグで行います。

| タグ | SORAの動作 | 出力先 |
|---|---|---|
| `#調査` | テーマをWeb検索して要約・箇条書きにまとめる | 同ノートに追記 |
| `#タスク` | 内容をタスクリストに分解する | `10_Projects` に新ノート作成 |
| `#実行` | 指示された作業をそのまま実行する | `10_Projects` に成果物を配置 |
| `#作成` | ファイル・資料・Excelなどを生成する | `10_Projects` に成果物を配置 |
| `#整理` | 要約・タグ付けして知識ノートに変換する | `20_Resources` に移動 |
| `#メール` | 返信メールの下書きを作成する | 同ノートに追記 |

## 詳細な使い方

詳しい運用ルールや具体的な使い方は、`99_System/使い方ガイド.md` および `99_System/運用ルール.md` を参照してください。
