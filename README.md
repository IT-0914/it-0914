# SORA: Cognitive OS for TAKUMI (v05.1)

SORAは、ユーザー「TAKUMI」専属の統合型コグニティブOSです。
戦略コンサルタント兼シニアPMのアイデンティティを持ち、Obsidianをインターフェースとして思考整理、意思決定、実務実行を一貫して支援します。

## 1. 動作原理
Obsidianのノートに特定のタグを付けてGitHubへpushすると、SORAが自動的に検知・処理を行い、結果をVaultに書き戻します。

- **Inbox保持**: 元ノートは `00_Inbox` から移動させず、永続的な思考の起点として保持します。
- **タグ隠蔽**: 処理済みのタグは `<!-- #タグ名 -->` に置換され、プレビュー上では非表示になります。
- **双方向リンク**: 成果物ノートと元ノートを相互にリンクし、トレーサビリティを確保します。

## 2. 実行スケジュール (365日稼働)
毎日（月〜日）の 8:00 から 20:00 まで、2時間おきに自動実行されます。**全ての実行回で `00_Inbox` のタグ処理が行われます。**
※ トークン消費最適化のため、変更がない場合は不要なファイル読み込みやGitHub同期をスキップします。

- **08:00 (朝)**: デイリーノート生成 ＋ タグ処理。
- **10:00 - 18:00 (日中)**: タグ処理（2時間おき）。
- **20:00 (夜)**: 活動ログの自動記入 ＋ タグ処理。

## 3. 対応タグ一覧
| タグ | 役割 | 出力先 |
|---|---|---|
| `#調査` | イベント報告・詳細レポート作成 | `20_Resources` |
| `#タスク` | プロジェクト・タスク分解 | `10_Projects` |
| `#実行` | 資料作成・実務遂行 | `10_Projects` |
| `#作成` | ファイル・アセット生成 | `10_Projects` |
| `#整理` | 思考構造化・知識化 | `20_Resources` |
| `#メール` | エグゼクティブチャット作成 | `00_Inbox` |
| `#指示` | 自由記述プロンプト実行 | 内容により判断 |
| `#週次レビュー` | 活動ログの戦略的総括 | `20_Resources` |
| `#緊急` | インシデント対応・指揮 | `10_Projects` |

## 4. ディレクトリ構造
- `00_Inbox`: 思考の起点・SORAへの依頼場所
- `01_Daily`: 自動生成されるデイリーノート
- `10_Projects`: 実行計画・プロジェクト管理
- `20_Resources`: 整理された知識・調査レポート
- `30_Archive`: 完了済みプロジェクト
- `99_System`: プロンプト・テンプレート・運用ルール
