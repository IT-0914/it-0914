# IT-0914 Obsidian Vault (v04)

TAKUMIの個人用Obsidian Vaultであり、AIエージェント「SORA」との共有ワークスペースです。

## 概要

このVaultは、戦略コンサル兼シニアPMのアイデンティティを持つSORAと連携し、知的生産を最大化するためのOSとして機能します。
TAKUMIが `00_Inbox` にメモを投げ、タグを付けてGitHubにpushすることで、SORAが自律的にタスクを処理し、構造化された成果物を書き戻します。

## 最新の運用フロー (v04)

1.  **Inbox保持**: TAKUMIが作成した元ノートは移動せず、常に `00_Inbox` に保持されます。
2.  **タグの隠蔽**: 処理が終わったタグは `<!-- #タグ名 -->` に置換され、視覚的なノイズを排除します。
3.  **1タグ1成果物**: 1つのメモに複数の依頼（タグ）がある場合、それぞれに対して独立した成果物ノートが生成されます。
4.  **双方向リンク**: 元ノートと成果物は相互にリンクされ、トレーサビリティが確保されます。

## SORAタグ一覧（全9タグ）

| タグ | 役割 | 出力先 |
|---|---|---|
| `#調査` | イベント報告・詳細レポート作成 | `20_Resources` |
| `#タスク` | プロジェクト・タスク分解 | `10_Projects` |
| `#実行` | 資料作成・実務遂行 | `10_Projects` |
| `#作成` | ファイル・資料生成 | `10_Projects` |
| `#整理` | 思考整理・構造化 | `20_Resources` |
| `#メール` | エグゼクティブチャット作成 | `00_Inbox` (追記) |
| `#指示` | 自由記述プロンプトの実行 | 内容により判断 |
| `#緊急` | インシデント対応・指揮 | `10_Projects` |
| `#週次レビュー` | 活動ログの戦略的総括 | `20_Resources` |

## フォルダ構成

- `00_Inbox`: 全ての新規メモ・依頼の入り口。
- `10_Projects`: 進行中のプロジェクト・タスク・成果物。
- `20_Resources`: 調査結果・整理済み知識・週次レビュー。
- `99_System`: 運用ルール・プロンプト・テンプレート。

詳細な運用ルールは `99_System/運用ルール.md` を参照してください。
